## 회의 내용

### 안건 1: 슬라이드형 PDF·한글 파일 활용
- **논의 내용**:
  - 슬라이드형 PDF와 한글(HWP) 파일을 함께 사용
  - 센서/모델 관련 설정도 포함해 전체 환경 세팅이 필요

### 안건 2: 기획서(PPT) 작성 및 타 멘토 피드백 계획
- **논의 내용**: 
  - 현재까지 기획한 내용을 PPT 발표자료라고 생각하고 정리해서 작성
  - 금요일쯤까지 PPT를 제출하고, 지금까지는 담당 멘토만 내용을 알고 있는 상태지만, 다른 멘토들도 PPT를 보고 피드백을 줄 수 있게 공유할 계획
  - PPT는 “발표자료+요약 기획서” 역할을 함께 하게 되므로, 전체 구조/흐름(문제의식–서비스 개요–기능–AI/RAG–인프라)이 잘 드러나야 함
  
### 안건 3: 모델 성능 테스트 및 요약 방식 논의
- **논의 내용**:
  - “Qwen2 12B가 1순위”라고 언급하며, 어떤 모델을 기준 모델로 둘지, 성능 테스트를 어떻게 설계할지 논의
  - 현 시점에서는 “Qwen2”가 적당한 결과를 내는 것으로 보임
  - 동일 질문/문서에 대해 모델별 응답 비교, 요약 품질, 응답 시간 등을 기준으로 한 간단한 벤치마크 계획이 필요

### 안건 4: RAGFlow 코드의 정의 및 보안 이슈
- **논의 내용**:
  - RAG 코드 구현에 시간이 걸리고 있음
  - RAG는 데이터 전처리가 끝나면 그 결과(텍스트/청크)를 넣고 한 번 테스트해볼 계획
  - RAG의 중요한 포인트는 청크를 어떻게 잘 분석하고 활용하는지(청크 분석 능력)이며, 문제는 RAG 자체라기보다, 다른 컴포넌트/플로우 쪽 테스트가 필요
  - RAG는 보안과 관련된 부분이 있어서, 실제 사내 파일을 RAG를 통해 온라인으로 처리하게 되면 데이터 유출 우려가 있다는 점을 지적
  - 내부/외부 경계, 마스킹 여부, 테스트용 더미 데이터 사용 등 보안 가이드라인에 맞는 RAG 테스트 전략이 필요

### 안건 5: 시험·오답노트·해설 보기 기능 및 규정안 적용
- **논의 내용**:
  - 시험 후 오답 노트 기능 구상: 시험을 보고 나서 “오답 노트” 버튼을 누르면 틀린 문제들이 리스트업되고, “해설 보기”를 누르면 사용자가 선택한 답, 정답, 해설 텍스트를 함께 보여주는 구조
  - 사용자의 사고/학습 맥락을 일정 시간 동안 유지하면서 피드백을 제공하고자 함
  - 새로운 규정안을 업로드하면, 해당 규정이 시험/해설 로직에도 적용되게 하는 것까지 염두에 두고 있음
  - 시험–정답–해설–규정 버전 간 연결(버전 관리, 적용 시점 등)을 어떻게 설계할지 고민 필요

### 안건 6: SSD 스토리지 용량 제한
- **논의 내용**:
  - 사용할 수 있는 용량이 100GB로 제한되어 있음
  - 파티션/용량 제안은 시스템 쪽에서 기본으로 걸려 있는 것으로 추정
  - 현재 사용량: 1조: 약 45GB, 2조: 약 35GB → 나머지 팀 포함하면 여유 공간이 그리 많지 않을 수 있음
  - 문서/데이터 저장 전략(불필요 파일 삭제, 압축, 예시 데이터 최소화 등)을 세워야 할 필요

### 안건 7: 로컬 연계 및 상품 보드 개발 계획
- **논의 내용**:
  - 로컬 환경 연계를 한 번 해보고, 최소한 “연결까지만” 되게 세팅한 뒤 멘토링 받을 예정
  - 멘토링이 끝난 후에는, “상품 보드(기능·할 일·우선순위 정리 보드)”에 개발할 것들을 정리하면서 구현을 이어갈 계획
  - 로컬 연계 PoC의 범위(어디까지 연동할지)와 상품 보드에 올릴 “개발 단위 카드” 정의가 필요

- **결정 사항**: 
  - PPT 발표자료: 현재까지의 기획 내용을 기반으로 발표용 PPT 초안을 작성하고, 금요일에 제출·피드백을 받는다 (기획) (2025-11-27~2025-11-29)
  - 모델 성능 테스트 계획: 우선순위 모델을 대상으로 요약/답변 품질·응답 속도를 비교하는 간단한 성능 테스트 계획을 수립하고, 테스트 문항·지표를 정의한다 (AI) (2025-11-27~2025-11-30)
  - RAG 코드·데이터 전처리: RAG 코드 테스트를 위해 사용할 문서/텍스트를 전처리·청크화하고, 보안 이슈를 피할 수 있는 테스트용/더미 데이터 세트를 구성한다 (AI / BE) (2025-11-27~2025-12-03)
  - 시험·오답노트 기능 설계: 시험–오답 노트–해설 보기 플로우와, 새로운 규정안을 업로드했을 때 시험/해설에 적용되는 룰(버전 관리 포함)을 요구사항 문서 형태로 정리한다 (기획 / FE / AI) (2025-11-28~2025-12-04)
  - SSD 스토리지 용량 관리: 스토리지(총 100GB) 사용 현황을 파악하고, 팀별 사용량(1조 45GB, 2조 35GB 등)을 기준으로 우리 팀이 사용할 수 있는 예상 용량과 관리 방안을 정리한다 (Infra / PM) (2025-11-27~2025-11-30)
  - 로컬 연계 PoC: 로컬 환경과의 연계를 “최소 동작 수준”까지 구현하고, 멘토링 시 데모 가능한 상태를 목표로 플로우와 연결 범위를 정리한다 (BE / AI) (2025-11-28~2025-12-03)
  - 상품 보드 정리: 멘토링 이후 진행할 개발 항목(기능 단위)을 상품 보드에 정리하고, 우선순위와 스프린트 배치를 함께 정의한다 (PM) (2025-11-28~2025-12-05)
