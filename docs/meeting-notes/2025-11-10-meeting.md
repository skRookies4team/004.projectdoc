## 회의 안건
1. Word Cloud API의 언어별 인식 문제
2. 코딩테스트(코테) 점수화 방식 및 처리 로직
3. 복지포인트 연계 시 부정방지 대책 수립

## 회의 내용

### 안건 1: Word Cloud API 문제점
- **논의 내용**:
  - 사용자가 영어와 한국어를 혼용하여 입력할 경우, 단어 빈도(counting)가 분리되어 정확도가 떨어짐
    - 예시: “security”와 “보안”이 같은 의미임에도 별도로 계산됨
  - AI 또는 NLP 모델을 활용해 동의어, 번역어를 통합 처리할 수 있는 방안 검토
  - 사전 전처리 단계에서 언어 통합 필터링 로직 추가 고려
  - 차별화 포인트로 “AI 기반 언어 통합 Word Cloud” 기능 제안 가능

### 안건 2: 코딩테스트(코테) 점수화 방식
- **논의 내용**: 
  - 단순히 정답 여부만으로 평가하면 공정성이 낮음
  - 코테의 본질은 “정확도 + 실행속도”임  
  - 테스트 케이스 전부가 아닌 샘플 데이터만으로 실행 결과를 평가하여 효율성을 확보
  - 점수화 방식은 **정답률(Accuracy)**과 **처리속도(Time Complexity)**를 가중합으로 산출하는 방안 논의
    - 예시: 최종점수 = (정답률 × 0.7) + (속도점수 × 0.3)
  - 초기 시범 단계에서는 샘플 데이터 기반으로 단순화하여 구현 후, 추후 확장 예정  

### 안건 3: 복지포인트 연계 및 부정방지 대책
- **논의 내용**:
  - 교육 이수나 테스트 결과를 복지포인트와 연계할 경우, 대리 시험·공동 응시·정답 공유 등 부정행위 가능성 존재
  - 로그인 세션 기반 사용자 식별 강화 (쿠키, 토큰 인증 활용)
  - 시험 중 화면 캡처/다중 접속 감지 기능 검토
  - 응시 중 브라우저 포커스 이탈 감지(Log Event Tracking) 기능 적용 가능성 논의
  - 포인트 부여 로직에 “부정행위 탐지 예외 처리 조건” 추가 필요
 
- **결정 사항**: 
  - Word Cloud: 영어-한국어 통합 처리 알고리즘 조사 및 적용 가능성 검토 (AI 파트) (2025-11-11)
  - 코테 점수화: 샘플 데이터 기반 속도·정확도 병행 점수화 로직 설계 (백엔드 파트) (2025-11-10)
  - 복지포인트: 부정행위 탐지 및 로그인 인증 방식 설계 (보안/FE 파트) (2025-11-10)
