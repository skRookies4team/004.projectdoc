## 회의 내용

### 안건 1: 중간발표 자료 작성 방향
- **논의 내용**:
  - 단순 중간점검용이 아니라 최종 발표 자료의 뼈대를 미리 만든다는 관점으로 접근해야 함
  - 전체 톤은 제품 홍보가 아니라 “프로젝트 제안/기획”에 가깝게, 문제 → 기존 방식의 한계 → 우리가 제안하는 교육과정/서비스 흐름으로 잡는 것이 좋음
  - 중간발표용 PPT = 최종발표 PPT에서 개요·목적·문제정의·MVP 연결 부분까지를 먼저 완성하는 느낌으로 설계해야 함

### 안건 2: 현황에서 문제 도출, 문제–MVP 연결고리
- **논의 내용**: 
  - 현재 현황(법정교육 진행 방식, 사내 LMS 운영, 직원 경험)에서 어떤 구체적인 문제들이 발생하는지를 먼저 정리해야 함
  - 이후, 각 문제와 MVP 기능 사이를 연결하는 “문제 ↔ MVP 기능 매핑 표(연결고리 표)”가 필요함
  - 큰 흐름 예시: 현재 현황” 슬라이드 → “현황에서 도출한 문제 리스트” → “각 문제를 어떤 MVP 기능으로 해결하는지 표로 정리” → 그 아래에서 개별 MVP 기능에 대한 기술·구현 설명 전개
  - 중간발표 전까지, 최소 1장 이상의 문제–MVP 매핑 표를 작성해야 함

### 안건 3: 작업 분담 및 기술 파트 정리 방식
- **논의 내용**:
  - PPT 작업량과 정리 난이도를 고려해, 주요 문서·슬라이드 작업은 두 명이 중심이 되어 진행
  - 인프라·MSA·AI 서버 같은 기술적인 내용은 담당자에게 맡기되, 담당자가 핵심 요약(1~2페이지 분량)을 제공하고 기획/문서 담당이 전체 흐름 속에 녹여 넣는 방식이 속도와 완성도 면에서 효율적
  - 역할 분담:
    - 기획/PM: 스토리·구조·문장 정리
    - 기술 담당자: 다이어그램, 키워드, 핵심 포인트 제공


### 안건 4: 로그인/Keycloak 이슈 및 우선순위 조정
- **논의 내용**:
  - 로컬 환경에서 프론트엔드를 실행했을 때, 로그인 페이지를 넘기고 싶지만 Keycloak에서 realm/클라이언트 설정(“리저 추가”)을 해도 아이디 패스 인증이 되지 않아 로그인/화면 전환이 되지 않는 이슈가 있음
  - 어드민 계정으로 들어가 계정/권한을 정리해 주는 방향으로 문제 해결을 시도
  - 현재 프로젝트의 메인은 그룹웨어 UI 전반이 아니라 “챗봇(어시스턴트)”이므로, 우선순위를 챗봇 기능 구현/시연 완성에 두고, 그룹웨어 페이지는 시간 여유가 생기면 보완하는 방향으로 조정
  - 중간·최종 발표에서 필요한 건 “최소한의 로그인/접근 흐름” + 챗봇 동작 데모이므로, 로그인 문제는 시연에 지장이 없을 정도까지만 해결해도 괜찮음

### 안건 5: MSA 요소 위치·로그 수집 역할
- **논의 내용**:
  - MSA 자체는 서비스 기능이 아니라 아키텍처 패턴에 가깝기 때문에, “MSA를 했다” 자체를 전면에 강조하기보다는, 서비스 분할 구조, 도메인 분리, 책임 분리를 보여주는 쪽이 더 나음
  - 로그 수집·분석 기능은 MSA의 “하나의 서비스”라기보다는 백엔드/운영 측면에서의 모니터링 기능으로 설명하는 것이 자연스러움
  - 발표 슬라이드에서 “MSA”는 아키텍처 다이어그램/기술 스택 일부로 조용히 녹이고, “로그/모니터링”은 품질/운영·리스크 모니터링 기능과 연결해서 설명하는 편이 좋음

### 안건 6: MSA 요소 위치·로그 수집 역할
- **논의 내용**:
  - AI 서버를 온프레미스 환경에 배포할 계획인데, 이것을 EKS 안에 둘지, 별도의 쿠버네티스 클러스터로 구성할지 애매한 상황
  - 결론적으로, 쿠버네티스 환경을 먼저 구성하고, 그 위에 AI 서버를 어떤 구조(EKS 포함 여부, 노드 구성 등)로 올릴지 설계해야 함
  - 인프라 설계가 한 차례 바뀌면서 전체 구조 정리가 다소 지연됨
  - 인프라/AI 파트는 “온프레미스 AI 서버 + 쿠버네티스/EKS 구조”를 하나의 아키텍처 다이어그램 슬라이드로 정리해 중간발표 PPT에 포함해야 함

- **결정 사항**: 
  - 중간발표 PPT 초안 제출: 중간발표용 PPT를 개요, 문제 정의, 문제–MVP 연결, 인프라 개략 구조까지 포함해 초안으로 작성 (기획 / PM) (2025-12-04~2025-12-05)
  - 문제–MVP 연결고리 표 작성: 현재 현황에서 도출한 문제 리스트와 각 문제를 해결하는 MVP 기능을 1:1 또는 1:N으로 매핑한 “연결고리 표”를 작성해 중간발표 자료에 포함 (기획 / 전체) (2025-12-05~2025-12-09)
  - 기술 파트 요약 인풋 수집: 인프라, MSA, AI 서버 관련 내용을 담당자가 정리하여 1~2페이지 분량의 핵심 요약(다이어그램 + 키워드)을 제공하고, 이를 발표 자료에 반영 (Infra / AI) (2025-12-05~2025-12-08)
  - 로그인/Keycloak 이슈 최소 해결: 시연에 필요한 최소 수준으로 로그인/권한 흐름이 동작하도록 Keycloak 설정(어드민 계정 정리 포함)을 조정 (BE /Infra) (2025-12-06~2025-12-12)
  - AI 서버 온프레미스·K8s 구조 설계도: AI 서버 온프레미스 배포를 전제로, 쿠버네티스/EKS 포함 여부를 정리한 인프라 아키텍처 다이어그램 초안을 작성해 중간발표 PPT에 반영 (Infra) (2025-12-07~2025-12-13)
