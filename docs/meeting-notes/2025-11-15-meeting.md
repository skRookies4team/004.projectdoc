## 회의 안건
1. 프로젝트 전체 방향성 재정립(서비스 중심 vs 기술 중심)
2. 교육 콘텐츠 자동 생성 시스템(영상 + 퀴즈) 구조 논의
3. 챗봇/어시스턴트 기능 및 RAG 적용 검토
4. 아키텍처 및 백엔드 구조 재설계 필요성
5. 교육 카테고리 및 직무교육 분류 기준 재정립
6. 콘텐츠 제작·승인 프로세스 정의
7. 데이터 처리/FAQ 전략 및 제외 범위 확정
8. UX/운영 정책 논의
9. 주요 산출물 및 제출 일정 점검

## 회의 내용

### 안건 1: 프로젝트 방향성
- **논의 내용**:
  - 프로젝트 목표는 AI 기반 교육 콘텐츠 자동 생성 + 학습 Assistant 개발
  - 서비스 중심·기술 중심 중 무엇에 초점을 둘지 결정 필요
  - 현재까지 방향성은 초기 기획과 크게 벗어나지 않음
  - 방향성이 확정되지 않으면 기능 범위가 계속 흔들릴 위험 존재
  - 기술 난이도 높은 요소(RAG·영상합성 등)와 서비스 안정성 사이 균형 필요

### 안건 2: 교육 콘텐츠 자동 생성 시스템
- **논의 내용**: 
  - 내부 문서(PDF/PPT) 기반 텍스트 검증 → 스크립트 → TTS → 영상 제작 흐름 논의
  - 3D 캐릭터 활용 가능성 언급
  - 자동 퀴즈 생성, 영상 종료 후 퀴즈 이동 흐름 설계
  - 텍스트 검증 과정은 사람이 일정 부분 직접 확인해야 함
  - 영상 합성 정확도 및 속도 측정 필요
  - 퀴즈는 교육마다 2회 제한, 점수 저장 로직 필요

### 안건 3: 교육 카테고리/분류 기준
- **논의 내용**:
  - 법정의무교육: 개인정보보호 · 성희롱 · 장애인 인식 · 직장 내 괴롭힘
  - 직무교육: NCS 기반 실시간 업데이트 가능
  - 보안 교육은 별도의 업무 교육이며 매우 다양한 주제로 구성되는데 하나의 동영상으로만 제공하는 것에 대한 문제 → ‘개인정보보호교육’을 ‘보안교육’이라고 잘못 표기하는 바람에 생긴 오해

### 안건 4: 챗봇/어시스턴트 기능
- **논의 내용**:
  - 단순 FAQ가 아닌 내부 문서 기반 RAG Assistant로 확장
  - 틀린 문제 관련 영상 재생, 규정 검색 등 교육 흐름 연동
  - 플로팅 아이콘 기반 챗 UI 고려
  - 질문 범위 정의 부족 → 검색 기능 품질 저하 우려
  - 대화 컨텍스트 유지와 토큰 비용 관리 기준 필요

### 안건 5: AI·RAG·데이터 처리
- **논의 내용**:
  - RAG(내부자료 기반) + LLM(외부지식 기반) 혼합 사용 전략
  - 벡터DB 온프레미스 구축 필요
  - 문서 표준화·벡터화 작업 난이도가 높아 개발 기간 내 완성도 이슈
  - “많이 나온 질문 패턴 분석 → FAQ 자동생성”은 난이도 및 기간 문제로 범위에서 제외

### 안건 6: 아키텍처 및 백엔드 구조
- **논의 내용**:
  - 시스템 아키텍처, 클라우드 아키텍처, DB 설계, 파이프라인 검토 필요
  - 백엔드 단일 서비스 구성은 적절하지 않음 → Mini-MSA 방향성
  - 기술 선택 시 근거 필요(보안, 구조, 성능 측면)
  - 벡터DB·LLM·AI 서버 구성에 따른 네트워크/보안 이슈 고려 필요

### 안건 7: 콘텐츠 제작·승인 프로세스
- **논의 내용**:
  - 콘텐츠 제작자 → 시스템 관리자 → 승인자(검수자) → 최종 사용자 구조
  - 영상 스크립트 검증, 품질 검수, 업로드 승인 필요
  - 이 프로세스는 각 팀이 ‘직무교육 콘텐츠를 직접 제작’할 경우 필수
  - 승인자 부재 시 품질 이슈·부적절 자료 업로드 위험

 ### 안건 8: UX 및 운영 정책
- **논의 내용**:
  - 영상 재생 UX: 이탈 시 자동 일시정지, 복귀 시 이어보기
  - 대화 기록 보관 정책 설정 필요
  - 챗봇 다크모드는 드롭다운 제거 과정에서 우선 제외됨

 ### 안건 9: 산출물 및 일정 관리
- **논의 내용**:
  - 2차 전까지 수행계획서 + 요구사항정의서 제출
  - 이후 인프라 구성도, 테스트 케이스, API 명세 등 작성 예정
  - 문서는 형식 제출이 아니라 실제 개발 기반이 되는 설계 문서여야 함

- **결정 사항**: 
  - 프로젝트 방향성: 서비스 중심 + 기술적 차별화 요소(RAG·영상 자동화) 병행 (차기 스프린트부터 고정) (2025-11-15)
  - FAQ 자동 생성 기능 제외: “패턴 분석 기반 FAQ 자동 생성”은 범위에서 제외, 이유: 사용자 질문 유사도 분석 작업·정형화 난도 높음 (명세서 반영) (2025-11-15)
  - 아키텍처 재설계: 백엔드 단일 컨테이너 구조 폐기, Mini-MSA 형태로 서비스 단위 분리 (다음 아키텍처 검토 회의까지(내주) 초안 마련) (2025-11-18)
  - 콘텐츠 제작·승인 프로세스 도입: 직무교육 콘텐츠를 직접 제작해야 할 경우
→ 제작–검수–승인 프로세스 필수 적용 (기능명세서·화면설계 반영) (2025-11-18)
  - UX 정책 확정: 영상 자동 일시정지/이어보기 적용, 챗봇 다크모드 삭제 (UI/UX 설계 시 즉시 반영) (2025-11-15)
