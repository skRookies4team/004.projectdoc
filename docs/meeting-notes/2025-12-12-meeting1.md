## 회의 내용

### 안건 1: AI 레포와 RagFlow 랩 분리·연결 구조
- **논의 내용**:
  - 기존에는 AI 레포를 RagFlow 관련 코드와 함께 사용하고 있었으나, 이제는 AI 레포는 AI 기능 전용, RagFlow 랩 레포는 문서 전처리·청킹·임베딩 전용으로 분리하는 방향으로 정리 중
  - AI 게이트웨이 관련 코드를 메인 브랜치에 강제로 반영해 둔 상태이며, RagFlow 랩 쪽 레포에는 기존 코드가 그대로 남아 있음
  - Git 브랜치 구성: 
    - RagFlow 관련 브랜치: 전처리(Preprocessing) 브랜치
    - AI 관련 작업 브랜치: AI 브랜치
    - 레포·브랜치 분리가 진행 중이므로, “AI ↔ RagFlow 랩 간 연동 포인트(입·출력 형식)”를 명확히 문서화할 필요 있음

### 안건 2: develop / main 브랜치, 리베이스 전략
- **논의 내용**: 
  - develop 브랜치를 main에 올리는 방식에 대해 논의:
    - develop을 main에 그대로 올려버리면 히스토리 상 main의 “축”이 흐려져 보이는 문제가 있어 시각적으로 좋지 않음
    - 따라서, main과 develop을 서로 한 번씩 거치며 싱크를 맞추는 방식이 필요함
  - rebase 사용 이유: rebase를 통해 커밋 히스토리를 재배치하여 관리자가 볼 때 변경 내역 흐름이 더 깔끔하게 보이도록 하기 위함
  - 결론: 기능 개발은 AI 브랜치 등 feature 브랜치 → develop → main 순으로 통합하고 필요 시 rebase를 활용해 시각적으로 정리된 히스토리 유지
  - Git 전략(merge vs rebase)을 팀 내 공통 규칙으로 정리해 두는 것이 필요

### 안건 3: 백엔드 코드 업로드·서비스 연결 현황
- **논의 내용**:
  - 현재 백엔드 코드가 원격 레포(main 기준)에 충분히 올라와 있지 않아, 프론트–백엔드–AI 전체 서비스 연결이 아직 완전하지 않은 상태
  - 프론트는 이미 main에 올리면서 화면 구조를 어느 정도 정리했으나,
서비스 단에서 실제로 호출할 백엔드 API 코드가 부족한 상황
  - S3 업로드 관련 백엔드 로직은 강사님이 이미 구현을 완료한 상태이며, 해당 부분은 PR로 가져다 써도 된다는 허가가 있었음
  - 브랜치·머지 상황: 브랜치를 정리한 게 아니라, 로컬로 가져와 정리한 뒤 PR을 통해 머지한 상태로 원격 기록 상 브랜치 이동이 명확히 보이지 않아 main에 직접 push한 것처럼 보이는 부분이 있어, 이력 관리에 주의 필요
  - 직접 main에 push하지 않고, 반드시 PR 경유하도록 백엔드 API 코드 정리 및 PR 전략을 다시 한 번 정리해야 함

### 안건 4: AI 브랜치 상태 확인 및 Git 히스토리 점검
- **논의 내용**:
  - AI 브랜치를 기준으로 develop과 main을 어떻게 맞물리게 했는지, 현재 브랜치 구조·히스토리가 어떻게 보이는지 모양을 확인 필요
  - 특히, PR 상에서는 분명 머지가 된 것처럼 보이는데, 로컬에서 새로 받아보면 변경 사항이 제대로 반영되지 않은 것처럼 느껴지는 상황이 있어서 이 부분에 대한 원인 파악 필요
  - Git 사용 시 pull 전략, rebase/merge 선택, main 보호 규칙 등을 정리해 같은 문제가 반복되지 않도록 할 필요가 있음

### 안건 5: 프라이빗 서버 앱 시연·온프레미스 연결 계획
- **논의 내용**:
  - 다음 주쯤 멘토님께 시연 영상을 보여드리는 것을 목표로 함
  - 시연 범위: 프라이빗 서버 앱 상에서 주요 화면·흐름이 동작하는 모습을 먼저 보여주는 것을 1차 목표로 설정
  - 현재는 외부 참고 자료(블로그 등)를 찾아보며 사이트/엔드포인트 연결 설정 방법을 검토 중이며, 온프레미스 환경과의 연결이 핵심 이슈로 남아 있음
  - 해당 연결 문제만 해결되면, 프라이빗 서버 앱에서 인터넷 트래픽을 어떻게 붙이고 라우팅할지까지 구체화할 수 있을 것으로 예상
  - 시연 시점까지 최소한의 온프레미스–외부 연결 플로우를 완성하는 것이 중요

- **결정 사항**: 
  - AI 레포·RagFlow 랩 역할 분리 문서화: AI 레포(모델·게이트웨이)와 RagFlow 랩 레포(전처리·청킹·임베딩)의 역할 및 연동 포인트를 정리해 문서화 (AI / 전처리)(2025-12-12~2025-12-18)
  - Git 브랜치 전략 정리: main–develop–AI(feature) 브랜치 전략, rebase/merge 사용 기준을 팀 공통 규칙으로 정리하고, 문서/Notion에 공유 (BE / FE) (2025-12-13~2025-12-19)
  - 백엔드 코드 정리 및 PR 진행: S3 업로드 등 이미 구현된 백엔드 기능을 포함해, 서비스 연동에 필요한 최소 API를 정리해 feature → develop → main 순으로 PR·머지 (BE) (2025-12-12~2025-12-22)
  - AI 브랜치 히스토리 점검: AI 브랜치에서 develop/main과의 싱크 상태 및 커밋 히스토리(리베이스 결과)를 점검해, 필요 시 브랜치 재정리 작업을 수행 (BE / AI) (2025-12-13~2025-12-20)
  - 프라이빗 서버 앱 시연 준비: 온프레미스 연결 이슈를 해결하고, 프라이빗 서버 앱에서 주요 화면·흐름이 동작하는 시연 영상을 제작해 멘토링 때 공유 (Infra / FE) (2025-12-14~2025-12-23)
