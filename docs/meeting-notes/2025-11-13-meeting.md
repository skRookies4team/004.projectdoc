## 회의 안건
1. 인프라 아키텍처와 기술 아키텍처의 구분 및 작성 방향
2. 시나리오 세분화 및 수정
3. 망 구성 및 보안 책임 체계 정립

## 회의 내용

### 안건 1: 인프라 아키텍처 구분 및 작성 방향
- **논의 내용**:
  - 인프라 아키텍처는 네트워크와 서버 물리적 연결, 망 구성, 접속 방식까지 포함해야 함
  - 서비스 아키텍처나 기술 아키텍처와 혼용되어 작성되고 있으므로 명확히 구분 필요
  - 백엔드 영역 요소를 인프라 구조에 적절히 배치해야 전체 구조가 완성됨
  - 현재 서비스 아키텍처 중심으로 되어 있어 인프라 시각화가 부족
  - 네트워크·보안 영역은 완벽한 수준이 아니라도 개념적 수준까지는 명시 필요

### 안건 2: 로드밸런싱 및 망 구성
- **논의 내용**: 
  - 로드밸런싱 적용은 서버 안정성 측면에서 유용하다는 의견
  - 폐쇄망/사내망 구성 개념만 이해해도 개발자 수준에서는 충분함
  - 인프라 아키텍처 초안은 별도 제공 예정
  - 보안 네트워크 구성과 접속 방식은 추후 세부 설계 시 보완

### 안건 3: 시나리오 세분화 및 수정
- **논의 내용**:
  - 기존 시나리오를 확장해 한 시나리오 안에서 여러 세부 흐름을 추가할 예정
  - 요구사항 정의서와 시나리오 간 매핑이 명확해야 함

### 안건 4: 망 구성 및 보안 책임
- **논의 내용**:
  - 회사 망을 직접 설계할 경우, 보안과 인증서 관리 책임을 명확히 해야 함
  - 내부 인증서, SSL 인증 절차 등 구체적인 보안 설계 항목을 인프라 문서에 포함해야 함
  - 온전한 내부망 구현 시, LLM도 내부망(Local) 환경에서 동작해야 함

### 안건 5: 온프레미스 vs 클라우드 환경
- **논의 내용**:
  - 온프레미스 방식은 보안 측면에서 유리하지만 구현에 많은 시간이 소요됨
  - 클라우드는 속도와 접근성이 좋으나 설정과 배포 과정에서 시간 소모가 큼
  - 핵심 목표는 클라우드 환경에서 AI 기능을 안정적으로 서빙하는 것
  - 인프라 선택은 프로젝트 기간 내 실현 가능성을 기준으로 판단

- **결정 사항**: 
  - 인프라 아키텍처: 서비스/기술 아키텍처와 구분된 인프라 아키텍처 초안 작성 (백엔드 / 인프라 파트) (2025-11-14)
  - 요구사항 매핑: 요구사항 정의서와 사용자 시나리오 간 매핑 (기획 / 프론트) (2025-11-13)
  - 보안 설계: 내부 인증서 및 SSL 인증 구조 작성 (인프라 / 보안) (2025-11-14)
