## 회의 안건
1. 개인화 기능의 우선순위 재조정
2. 복지·평가·근무 정보 기반 개인화 리포트 구상

## 회의 내용

### 안건 1: 개인화 기능의 우선순위 재조정
- **논의 내용**:
  - 개인화 관련 논의가 늦게 본격화되어 처음 기획에서는 우선순위가 낮게 잡혀 있었음
  - 다른 조(개인화 중심 주제 팀)와 겹칠까 봐 개인화를 뒤로 미뤘으나, 이번 프로젝트에서 개인화는 “반드시 가져가야 할 핵심 축”이라고 강조하는 멘토 의견
  - 백엔드 코드·보안(계정/환경변수) 정리
  - 기존 기획서·발표 자료에서 개인화가 부가 기능처럼 표현된 부분은 핵심 가치로 올라오도록 재구성 필요

### 안건 2: 복지·평가·근무 정보 기반 개인화 리포트 구상
- **논의 내용**: 
  - 개인화의 구체적인 예시로, 복지, 평가, 근무 형태(연차·야근·추가 수당 등)를 한눈에 볼 수 있는 “개인 상태창” 콘셉트가 제안됨
  - RPG 게임의 상태창처럼, “내 근무 현황 알려줘”라고 치면 남은 연차, 근무·야근 시간, 추가 수당, 평가에 반영될 만한 항목들을 정리된 리포트 형태로 보여주는 화면을 목표로 하자는 의견
  - 출력 형태는 리포트형 화면 또는 챗봇이 정리된 내용으로 답변해 주는 형식 둘 다 가능
  - 최종 산출물 관점에서 “직원 개개인의 HR·복지·평가 정보를 모아주는 개인화 리포트”가 메인 스토리/핵심 데모 포인트가 될 수 있음

### 안건 3: 백엔드 코드·보안(계정/환경변수) 정리
- **논의 내용**:
  - 백엔드 코드에서 아이디·패스워드 정보가 Git 레포에 올라간 것처럼 보이는 부분이 있다는 지적
  - 팀 내 논의 결과: 환경 변수(env)나 계정 정보는 레포에서 제거하고, 다소 번거롭더라도 별도 파일(로컬/비공개 공유)로 관리하는 방향이 맞음
  - 현재는 API 연동도 백엔드에서 외부/API 콜 부분이 아직 본격 구현되지 않은 상태
  - 보안·운영 관점에서 비밀 정보 레포 노출 방지 및 백엔드 API 레이어 설계·구현 일정 확보가 필요

- **결정 사항**: 
  - 개인화 전략 재정의: 기존 “교육·퀴즈 중심” 개인화에서 확장해, 복지·평가·근무 정보까지 아우르는 개인화 핵심 시나리오(개인 상태창/리포트)를 정의하고 기획서·PPT에 반영 (기획 / PM)(2025-12-16~2025-12-22)
  - 개인화 리포트 화면 설계: “내 근무 현황 알려줘” 요청 시 연차, 근무·야근, 추가 수당, 평가 포인트 등을 한 화면/챗봇 답변으로 보여주는 UI·플로우 와이어프레임을 설계 (기획 / FE) (2025-12-16~2025-12-24)
  - 관리자 기능 범위 조정: 관리자 대시보드는 최소 기능 위주로 축소하고, 개인화 기능을 메인으로 가져가는 구조로 요구사항·기획 문서를 수정 (기획) (2025-12-17~2025-12-23)
  - 온프레미스 필요성 검토: 온프레미스 환경을 꼭 써야 하는 이유(보안·비용·데모 환경)를 정리하고, 순수 클라우드 대안과 비교해 아키텍처 방향을 확정 (Infra / PM) (2025-12-18~2025-12-27)
  - 백엔드 보안 정리: Git 레포에서 아이디·패스워드 등 민감 정보 및 환경 변수를 제거하고, .env 등 별도 비공개 파일로 재구성한 뒤 팀 내 공유 원칙을 문서화 (BE) (2025-12-16~2025-12-19)
  - API 연동 계획 수립: 현재 미구현 상태인 백엔드→외부/내부 API 콜 구조를 설계하고, 우선 구현할 엔드포인트 목록과 일정(WBS)을 정리 (BE / AI) (2025-12-19~2025-12-26)
