## 회의 내용

### 안건 1: 중간발표 PPT 대상 및 개요 구성 방향
- **논의 내용**:
  - 이번 중간발표 PPT의 주요 대상은 멘토들
  - 프로젝트 개요를 기존보다 더 구체적으로 구성해야 함
  - 문제 제기
    - 4대 법정의무교육 등 법정교육을 실제로 수행하는 회사들에 대한 타깃 조사, 불편한 점
    - 소비자(사용자) 니즈
    - 기존 시스템의 문제 해결
  - 단순 기능 소개가 아니라, “누가 어떤 문제로 불편해하고 있고, 그걸 왜 우리가 해결하려고 하는지”가 명확히 드러나야 함

### 안건 2: 문제 정의와 핵심 아이디어 연결
- **논의 내용**: 
  - 현재 ‘문제’와 ‘핵심 아이디어’는 제시되어 있으나, 두 요소 사이의 논리적 연결고리가 약하다는 인식 공유
  - “문제 → 그 문제를 어떻게 풀 것인지(핵심 아이디어)”를 한 번 더 ‘자세히 디벨롭(구체화)’해야 한다는 의견
  - 특히, 법정교육/사내교육의 현재 운영 방식에서의 문제점과 그 문제를 해결하기 위해 우리가 제안하는 AI 어시스턴트·교육 추천·퀴즈·모니터링 등 핵심 아이디어를 자연스럽게 이어주는 구조가 필요
  - 중간발표 슬라이드에서 “문제 정의 슬라이드”와 “핵심 아이디어 슬라이드”가 이야기처럼 연결되도록 재구성해야 함

### 안건 3: 목표 설정: 기술적 목표와 서비스적 목표
- **논의 내용**:
  - 프로젝트 목표는 두 축으로 나누어 정리하기로 함
    - 기술적인 목표: “우리가 어떤 기술적 시도를 하는 프로젝트인지”를 명확히 표현
    예시: RAG, 벡터 DB, 롤 기반 접근, 마스킹, 전처리 파이프라인 등
    - 서비스적 목표 (고객 관점 목표): 사용자인 직원/교육담당자/관리자에게 어떤 가치를 제공할 수 있는지 정리
  - 서비스적 목표에 포함되는 키워드
    - 부정 행위 방지(대리 시험, 정답 공유 등) → 비전 측면 목표
    - 모니터링: 조직/부서/교육 이수 현황, 리스크 등을 어느 정도까지 정형화해서 보여줄 수 있을지 검토 필요
    - 안심하고 질문할 수 있는 환경: 마스킹 처리를 통해 마음편히 물어봐도 개인정보보호가 됨
  - “기술적으로 무엇을 하겠다”와 “사용자/기업에 어떤 효용을 준다”를 PPT와 문서에서 명확히 분리해서 표현해야 함

### 안건 4: 수행절차 및 팀 자체 규칙 정리
- **논의 내용**:
  - ‘수행절차(프로세스)’는 WBS를 통해 전체 흐름이 보이도록 정리하기로 함
    기획 → 데이터 수집/전처리 → 개발 → 테스트 → 평가 등의 큰 흐름
  - 이 과정에서, 우리 팀이 스스로 세운 규칙을 하나의 강점으로 보여줄 수 있음
    예시: 요구사항명세서 ID 부여 기준
    도메인별로 나누고, 각 도메인 페이지마다 필요한 기능을 1차적으로 정리한 뒤, 그 구조를 기준으로 요구사항 ID를 부여한 방식
  - 이는 단순히 문서 형식이 아니라, 우리 팀의 개발 방법론·설계 접근 방식의 일부로 설명 가능
  - WBS + 팀 내 규칙(요구사항 ID, 도메인 구조 등)을
“우리 팀의 일하는 방식”으로 보여줄 수 있도록 문서화 필요

### 안건 5: 아키텍처·인프라·AI 서비스 플로우 및 롤별 접근
- **논의 내용**:
  - 아키텍처 구성도, 인프라 구성도, AI 서비스 전체 플로우 정리
  - 롤(관리자, 교육담당자, 일반 직원 등)에 따라 어떤 화면·기능에 접근할 수 있는지(롤별 접근)를 제시
  - 아키텍처/플로우는 단순한 기술 나열이 아니라 “누가 어떤 경로로 어떤 기능을 쓰는지”까지 보이도록 구성해야 함

- **결정 사항**: 
  - 문제·개요 슬라이드 재구성: 4대 법정교육 및 사내교육의 현황·불편함, 소비자 니즈, 기존 시스템 문제와 핵심 아이디어를 하나의 스토리로 연결해 개요 슬라이드를 재작성 (기획) (2025-12-03~2025-12-09)
  - 기술적·서비스적 목표 문서화: 기술적 목표(기술 스택, 시도하는 요소)와 서비스적 목표(부정행위 방지, 모니터링, 보안/마스킹)를 구분하여 목표 정의 문서를 작성하고 중간발표 자료에 포함 (기획 / AI) (2025-12-03~2025-12-10)
  - WBS 및 팀 규칙 정리: 전체 수행절차(WBS)와 팀이 스스로 정한 규칙(요구사항 ID 부여 기준, 도메인별 기능 정리 방식)을 정리해 중간발표 자료에 포함 (PM / 기획) (2025-12-03~2025-12-11)
  - 아키텍처·플로우·롤별 접근: 아키텍처 구성도, 인프라 구성도, AI 서비스 플로우 중간발표 자료에 포함 (Infra / AI / FE) (2025-12-05~2025-12-13)
