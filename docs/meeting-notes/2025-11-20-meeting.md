## 회의 안건
1. AWS 아키텍처 구성 시 사양·비용 산정 방식 점검
2. AWS 공식 문서 및 가이드라인 기반 아키텍처 이해
3. 보안 프로젝트에서 내부 통신 구조 검토
4. 인프라 구성(단일 서버 vs 분할 서버) 비교
5. 내부 개발용 아키텍처 문서 정리 방향 및 공유 방식 검토

## 회의 내용

### 안건 1: AWS 사양/비용 산정 방식
- **논의 내용**:
  - AWS 사용량 및 사양 계산 시 AWS 요금 계산기를 활용해야 함
  - 인스턴스 타입 2개를 사용할 경우 요금 계산도 타입별로 수행해서 2회로 계산해야 함
  - 사양(Spec)은 기술 구조, 규격, 용도에 맞춰 정확히 기재해야 함
  - 과금 방식(AWS On-demand, Reserved 등) 선택에 따라 총 비용이 크게 변동됨
  - 아키텍처 전체 구조 설명은 이후면담에서 다시 설명 가능

### 안건 2: AWS 아키텍처 설계 시 유의점
- **논의 내용**: 
  - 클라우드는 6개월 단위로 대규모 업데이트가 이루어지므로 오래된 PPT 구조는 빠르게 의미가 없어짐
  - AWS 공식 가이드라인을 기반으로 아키텍처 설계 권장
  - 발표용 PPT는 너무 복잡하게 그리기보다 단순화·요약된 형태로 만드는 것이 적절함
  - 인사 담당자 등 비전공자도 이해할 수준으로 간단한 구조가 필요
  - PPT에 과도한 서비스 나열은 전문성 검증이 불가능하게 보일 수 있음
  - 발표용 아키텍처는 "본질만 담긴 단순화된 그림"이어야 평가에 유리함
  
### 안건 3: AWS 공식 문서 검토 중요성
- **논의 내용**:
  - AWS는 서비스별 공식 문서에 서비스 개념·사용법·구축 방식·활용 사례가 매우 자세히 정리되어 있음
  - 새로운 서비스나 생소한 기능을 사용할 때는 반드시 공식 문서를 확인해야 함
  - 외부 블로그나 영상은 오래되었거나 잘못된 정보가 포함될 수 있으므로 아키텍처가 왜곡될 수 있음

### 안건 4: 보안 관련 프로젝트의 내부 통신 문제
- **논의 내용**:
  - 현재 기획한 구조대로라면 ‘내부 시스템 간 통신이 막힐 가능성’이 있다고 조언
  - 특히 보안·폐쇄망 환경에서는 외부로 데이터를 보내기 어렵기 때문에 외부·내부를 오가며 통신해야 하는 경우, Direct 연결 방식(전용선/전용 통신 경로) 사전에 고려 필요
  - 사내망 또는 폐쇄망 환경에서 LLM·API 연동이 필요한 경우 LLM 호출이나 API 요청을 내부에서 처리할지 외부에서 처리할지 구조적 판단 필요

### 안건 5: 인프라 관점: 단일 서버 vs 분할 서버
- **논의 내용**:
  - 실무 관점에서 단일 서버 구성도 안정성 측면에서 큰 문제없음
  - 서버를 분할 구성해도 실질적인 안정성 상승은 크지 않을 수 있음
  - 분할 아키텍처는 개발 복잡도·네트워크 구성 부담 증가
  - MVP 단계에서는 단일 서버 중심 구성이 더 효율적

### 안건 6: 내부 개발 환경 공유 및 아키텍처 검증
- **논의 내용**:
  - 팀이 내부 개발용 아키텍처를 보여줬을 때 “이게 어디에 포함되는 구조인가?”라는 질문이 나옴
  - 중간발표에서도 비슷한 지적이 있었음
  - 정식 아키텍처 파일을 함께 제공할 수 있도록 개발 파일의 저장·공유 구조 개선 필요
  - 내부 전용 아키텍처 문서를 별도로 보관하지 않으면 내부에서만 보고 있는 구조와 공식 발표용 구조가 따로 노는 문제가 발생하여 발표 때 혼란이 발생함

- **결정 사항**: 
  - AWS 아키텍처: 요금 계산 및 사양 정리 후 PPT 초안 작성 (Infra/BE) (2025-11-22~2025-11-25)
  - 공식 문서 활용: AWS 공식 문서 기반 서비스 이해 및 문서화 (Infra) (2025-11-21~2025-11-22)
  - 보안 통신 구조: 폐쇄망/보안 환경에서의 Direct 통신 가능성 추가 검토 (Infra/Security) (2025-11-23~2025-11-27)
  - 인프라 구성: MVP는 단일 서버 기반 아키텍처로 설계 (PM/BE) (2025-11-21~2025-11-22)
  - 아키텍처 문서: 정식 아키텍처 PDF/PPT 별도 저장 및 공유 (팀 전체) (2025-11-22~2025-11-24)
