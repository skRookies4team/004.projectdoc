## 회의 안건
1. 관리자 페이지 또는 대시보드 내 리스크 모니터링 및 경영자 보고 기능 누락 문제
2. 차별화 전략 추가 방안 논의 (UI/UX, 데이터 정확도, 보고서 시각화 등)
3. 서비스 제작 배경 구체화 및 문서 반영

## 회의 내용

### 안건 1: 관리자 페이지 / 대시보드 리스크 모니터링 기능 누락
- **논의 내용**:
  - 현재 기획안 및 화면 설계서에 경영자용 리스크 모니터링 기능이 포함되지 않음
  - 어시스턴트가 수집한 대화/교육/신고 데이터를 기반으로 리스크 트렌드나 통계 정보를 경영진에게 시각적으로 보고하는 구조 필요
  - 관리자 페이지 또는 대시보드 내 “리스크 모니터링 탭” 추가 제안
  - 주요 기능 예시:
    - 부서별·주제별 리스크 발생 빈도 시각화 (Word Cloud / 그래프)
    - 분기별 주요 리스크 유형 리포트 자동 생성
    - 관리자 이메일·대시보드로 주기적 보고
  - 보고 시각화 자료는 Chart.js, Recharts 등 활용 가능

### 안건 2: 차별화 전략 보완
- **논의 내용**: 
  - 서비스가 단순한 교육 안내형 어시스턴트로 인식될 가능성 있음
  - 사용자가 아이콘을 클릭하고 싶게 만드는 UX 중심 설계 필요
  - 단순히 데이터 정확도나 교육 콘텐츠 품질 외에도, 시각적으로 흥미로운 리포트·대시보드 표현이 차별화 요소가 될 수 있음
  - 예시:
    - 결과 보고서를 단순 텍스트 대신 인터랙티브 카드, 애니메이션 그래프 등으로 표현
    - 사용자 참여도를 높이는 “게임화(Gamification)” 요소 도입 가능
    - UI 디자인 시 “호기심 유발형 인터페이스”를 테마로 기획할 것

### 안건 3: 서비스 제작 배경 구체화
- **논의 내용**:
  - 기존 문서에 서비스 제작 배경이 간략히만 언급되어 있음
  - 시장 현황: 현재 기업의 내부교육 서비스는 메신저형(대화형) 또는 단편적 콘텐츠 제공 형태로 분산되어 있음
  - 제작 배경: 이러한 단편적·비연계형 구조를 개선하고, 기업 맞춤형으로 통합적이고 지능화된 어시스턴트형 교육·리스크 관리 서비스를 제공하기 위함
  - “고도화된 내부교육 및 리스크 인텔리전스 어시스턴트 제공”이라는 문장으로 요약 반영

- **결정 사항**: 
  - 리스크 모니터링: 관리자 페이지에 리스크 모니터링 탭 신설 및 시각화 구조 설계 (백엔드 / 데이터 파트) (2025-11-12)
  - 차별화 전략: UX 중심 대시보드/리포트 디자인 아이디어 수집 및 반영 (프론트엔드 / 기획) (2025-11-12)
  - 제작 배경: 프로젝트 수행계획서 및 제안서에 배경 서술 보완 (기획 / PM) (2025-11-12)
