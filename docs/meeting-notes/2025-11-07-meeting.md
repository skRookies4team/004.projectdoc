## 회의 안건
1. 프로젝트 차별화 포인트 도출 및 명칭 확정
2. 요구사항 정의서(Use Case Diagram, 시나리오) 작성 방향 논의
3. 어시스턴트 UI 진입 구조 및 인증 설계 방향 논의

## 회의 내용

### 안건 1: 차별화 요소 구체화
- **논의 내용**:
  - 단순한 교육 챗봇이 아니라,
    → 직원 질의 데이터를 분석해 **경영진에게 리스크 인사이트를 제공**하는 기능 제안
  - 예시:
    - 성희롱 관련 질문이 특정 부서에서 많이 발생 → 부서별 리스크 요인 리포팅  
    - 문의/이슈 발생 빈도를 기반으로 리스크 트렌드 리포트 생성  
  - 이러한 데이터 기반 인사이트는 **경영진 보고용 리스크 대시보드**로 확장 가능  
  - 해당 아이디어는 **노션 기획 문서에 추가 완료**

### 안건 2: 요구사항 정의서 작성 방향
- **논의 내용**: 
  - 각각에 요구사항 ID를 부여하여 개발자 관점에서 Use Case Diagram을 기반으로 시나리오별 상세 기술 필요
  - Use Case 내용, 사전조건, 사후조건, 기본흐름, 예외흐름, 대안흐름에 대한 시나리오 작성  

### 안건 3: 어시스턴트 UI 및 진입 구조
- **논의 내용**:
  - UI 형태: 그룹웨어 내 **Floating Assistant 아이콘** 형태로 구현 
    - 아이콘 클릭 시 → Card 또는 Workflow 창이 팝업
  - **사용자 진입장벽 고려:**  
    - 이미 그룹웨어 로그인 상태라면 **별도 로그인/회원가입 페이지 불필요**
    - 그룹웨어 인증 기반으로 **쿠키/토큰만 받아 인증 처리**
  - **기술 참고:**  
    - `Keycloak` 등의 인증 서버를 통해 그룹웨어 로그인 정보 검증 가능  
    - 인증 성공 시 토큰 발급 → 어시스턴트가 해당 세션에서 작동
  - UI 확장 아이디어:
    - 가상의 그룹웨어 대시보드(백그라운드)를 구성  
    - 어시스턴트 창에서 교육 페이지, 규정 페이지 등으로 직접 연동  
    - 백그라운드 화면에 해당 페이지가 표시되도록 연동  식 대응
