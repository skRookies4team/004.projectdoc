## 회의 내용

### 안건 1: MSA 온프레미스·AWS 환경에서의 게이트웨이 필요성
- **논의 내용**:
  - MSA 환경(온프레미스, AWS 모두)에서는 여러 서비스의 경로를 찾아주는 게이트웨이가 필수라는 점을 재확인
  - AWS 환경에서는 비교적 제공 요소들이 있어 구현이 수월하지만, 온프레미스 환경에서는 우리가 별도의 게이트웨이 역할을 하는 컴포넌트/설계를 마련해야 한다는 논의
  - 향후 아키텍처 다이어그램에 ‘게이트웨이 레이어(역할, 위치, 트래픽 흐름)’를 명확히 표현할 필요가 있음

### 안건 2: AIOps, 전처리 기간·과정 정리
- **논의 내용**: 
  - AIOps 관점(실제 변동 사항을 사람이 간섭하지 않고 어떤 특정 규칙에 맞게끔 넘기면 스스로 AI가 점수 처리하고 데이터에서 다시 학습해서 RAG 데이터를 갱신하거나 모델을 화이트닝 시키는 것)에 대해서도 생각해봐야 함
  - 데이터 전처리에 걸리는 기간을 가늠해 보고, 전처리 과정 자체를 정리/가시화하라는 요청이 있었음
  - 아키텍처 + 전처리 흐름을 한눈에 볼 수 있는 자료를 준비해야 함
  
### 안건 3: 비정형 데이터 분류 및 시각화 데이터 추출
- **논의 내용**:
  - 비정형 데이터의 특성상 단일 기준으로 일괄 정규화하는 데에는 한계가 있으며, 분석 목적과 품질 기준에 따라 일부 데이터는 과감히 제외하는 정제 전략이 필요하다는 의견이 제시됨
  - 실제로 우리가 어떻게 비정형 데이터를 분류했는지, 그 결과를 시각화 데이터로 어떻게 추출하는지까지 보여주는 것만으로도 충분히 의미가 있음
  - RAG/전처리성과를 보여줄 때, ‘정답률’뿐 아니라 분류·시각화 플로우 자체를 보여주는 데모도 강점이 될 수 있음

### 안건 4: 자료 제공 범위 조정
- **논의 내용**:
  - 사내 프로세스, 계획, 매뉴얼로 문서의 범위 조정 및 승인 후 제공 가능

- **결정 사항**: 
  - MSA 게이트웨이 설계 검토: 온프레미스·AWS 환경 각각에 대해 필요한 게이트웨이 역할(라우팅, 인증 등)을 정리하고, 아키텍처 다이어그램에 반영할 초안을 작성한다 (Infra / BE) (2025-11-28~2025-12-03)
  - 전처리 기간·프로세스 정리: 비정형 데이터 전처리에 걸리는 예상 기간을 산정하고, 전처리 과정(수집–정제–청크–라벨링)을 단계별 플로우/다이어그램으로 정리한다 (AI / 기획) (2025-11-28~2025-12-04)
  - 비정형 데이터 시각화 PoC: 현재 분류 기준을 바탕으로, 비정형 데이터를 어떻게 분류·집계·시각화할지 간단한 예시 화면(또는 그래프)으로 PoC를 진행한다 (AI / FE) (2025-11-29~2025-12-05)
