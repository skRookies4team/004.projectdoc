## 회의 내용

### 안건 1: 관리자 프로그램 기반 문서 업로드·변환 프로세스
- **논의 내용**:
  - 사용자는 관리자 프로그램에서 문서를 업로드
  - 관리자 프로그램이 오피스 문서(한글, PPT 등)를 PDF로 자동 변환
  - 변환된 PDF가 시스템에 업로드되어 이후 전처리·RAG 파이프라인으로 전달되는 구조로 정리
  - 업로드 → 변환 → 저장까지의 흐름을 하나의 관리 화면/시나리오로 정리할 필요

### 안건 2: 이미지·텍스트 OCR 인식 문제
- **논의 내용**: 
  - 현재 이미지 안의 텍스트를 OCR로 인식할 때 글자가 깨지는 등 인식 품질이 좋지 않은 문제가 있음
  - 이미지를 포기했다는 것은 이미지 OCR을 완전히 버린다는 의미가 아니라 텍스트 PDF처럼 동일한 방식으로 처리하기는 어렵다는 의미에 가까움
  - “이미지 안의 OCR를 읽으라”는 요구는 없었고, 이미지는 이미지대로 별도 처리 전략이 필요하다는 점을 재확인
  - 이미지 기반 자료는 OCR 품질이 허용 가능한 범위인지 재검토하는 것과 아예 텍스트 위주 컨텐츠에 초점을 맞추는 것에서 선택 필요

### 안건 3: 도메인별 FAQ 설정·화면 구성·유사도 검색
- **논의 내용**:
  - 현재 채팅 화면은 데이터셋과 완전하게 연결되지는 않은 상태이고, 다음 주 월요일에는 실제 연결된 모습을 보여줄 수 있을 것으로 예상
  - 도메인별 FAQ는 관리자가 수작업으로 FAQ 후보를 선택·등록하는 방식으로 설계
  - 도메인 UI:
    - 기본적으로 메인 화면에서 도메인이 칩 형태로 노출되어, 클릭 시 해당 도메인 화면으로 이동
    - FAQ 화면은 상단 헤더에 도메인 선택 영역을 두고, 도메인 선택에 따라 FAQ 리스트가 달라지도록 설계
  - 유사도 검색:
    - 이전에는 전체 데이터에 대해 유사도 검색(top-k) 를 하다 보니 엉뚱한 FAQ가 노출되는 문제가 있었음
    - 개선 방향으로 “도메인별로 먼저 묶은 뒤, 그 안에서만 유사도 검색”을 수행하는 방안을 논의
    - 필요 시 유사도 임계값 조정 또는 채팅창/FAQ 노출 방식 변경도 고려
  - 도메인 필터링 → 유사도 검색 순서로 바꾸는 설계를 FAQ·챗봇 공통 로직으로 정리할 필요

### 안건 4: 시스템 관리자용 챗봇 및 로그 저장
- **논의 내용**:
  - 시스템 관리자용 챗봇 화면을 별도로 구현해 두었으며, 일반 사용자용 챗봇과 역할/권한을 구분하는 구조
  - 대화 로그는 DB에 전부 저장되는 방식으로 설계되어 있음(저장 위치: DB 대화 테이블)
  - 어떤 로그를 얼마나 오래 어떤 목적으로 보관할지에 대한 로깅·보관 정책 정의 필요(개인정보·감사 목적 등 고려)

### 안건 5: AI 서버 역할·개인화 범위·개인정보 보호
- **논의 내용**:
  - AI 서버는 백엔드 기능 전반과 연결된 통합 영역으로 보고 있으며, 여기서 생성되는 채팅 답변·교육 영상 등을 별도로 보관할지에 대해 논의
  - 개인화 범위:
    - 교육 퀴즈, HR 근태, 복지 포인트 등을 개인화 대상 영역으로 설정 필요 시 유사도 임계값 조정 또는 채팅창/FAQ 노출 방식 변경도 고려
    - 이때 개인화 데이터가 외부로 나가지 않도록 하는 것이 핵심 원칙
  - 롤·페르소나: 관리자 등 롤(role)에 따라 다른 답변을 주는 것보다는, 페르소나(역할별 챗봇 프로필) 단위로 분리하는 접근이 더 자연스러울 수 있음
  - AI 응답 특성: 현재 모델은 “AI가 말하는 것처럼” 답을 주는 수준이고, “특정 사람이 말하는 것처럼” 자연스럽게 만들려면 그 사람에 대한 충분한 데이터·학습이 필요하다는 한계가 있음
  - 개인화 + 로그·영상 보관이 결합될 경우, 개인정보·내부감사 관점에서 어떤 정보까지 남길지 기준 설정이 필요

- **결정 사항**: 
  - 문서 업로드·변환 플로우 정리: 관리자 프로그램에서 업로드 → 오피스→PDF 변환 → RAG 전처리까지의 전체 흐름을 시나리오/다이어그램으로 정리 (전처리 / BE) (2025-12-15~2025-12-18)
  - 이미지·OCR 처리 정책 확정: 이미지 내 텍스트는 어느 수준까지 OCR로 처리할지, 또는 텍스트 기반 문서 위주로 한정할지에 대한 정책을 정리 (전처리 / AI) (2025-12-15~2025-12-20)
  - 도메인별 FAQ·유사도 검색 로직 구현: 도메인 필터링 후 유사도 검색을 수행하는 구조로 FAQ 검색 로직을 설계·구현하고, FAQ 관리 화면(도메인·질문 선택)을 보완 (BE / FE) (2025-12-15~2025-12-22)
  - 관리자용 챗봇 로그·보관 정책 정의: 시스템 관리자용 챗봇의 대화 로그 저장 위치·보관 기간·조회 권한 등 로깅 정책을 정리하고 DB 스키마에 반영 (BE / 기획) (2025-12-15~2025-12-23)
  - 개인화 범위·보안 원칙 문서화: 개인화 대상(교육 퀴즈, HR, 복지)과 비개인정보 외부 비유출 원칙, 마스킹 범위를 문서로 정의하고, AI 서버의 답변·영상 보관 여부 기준을 함께 정함 (기획 / AI) (2025-12-16~2025-12-24)
